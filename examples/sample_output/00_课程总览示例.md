# 分布式数据库系统 期末复习课程总览（示例）

## 1. 这门课到底学什么

分布式数据库系统研究的是：当数据不再集中存放在单一机器上，而是分布在多个站点或节点中时，数据库系统如何仍然向用户提供统一、可靠、高效的数据管理能力。

这门课的核心不是只记住几个定义，而是理解一条主线：数据如何分布、查询如何执行、事务如何保证正确性、故障后如何恢复、多副本之间如何保持一致。

## 2. 一句话学习目标

> 学完这门课，本质上是要理解：在数据分布于多个节点的情况下，数据库系统如何实现透明访问、正确事务、高效查询和可靠恢复。

## 3. 课程核心问题链

```text
为什么需要分布式数据库
→ 数据如何分片和复制
→ 用户如何透明访问分布式数据
→ 查询如何跨站点执行和优化
→ 事务如何在多个节点上保持 ACID
→ 故障后如何恢复
→ 多副本如何保持一致
```
