# Workflow Reference

本文件定义 Final Exam Course Rebuilder Skill 的标准执行流程。

## 0. 接收输入

用户通常会提供一组 PDF/PPT/PPTX 课件。可能还会提供：

- 课程名
- 考试时间
- 考试大纲
- 教师复习提纲
- 往年题或样题
- 用户自己的笔记

若关键信息缺失，不要停在澄清问题上。除非用户明确要求先确认，否则应基于已有材料先生成可用结果，并将假设写明。

## 1. 建立文件清单

先建立文件清单，用于掌握材料范围。

推荐字段：

| 文件名 | 类型 | 页数/幻灯片数 | 推测章节 | 主要主题 | 重要图示/例子 | 备注 |
|---|---|---:|---|---|---|---|

## 2. 全局阅读

全局阅读所有材料，目的不是逐文件摘要，而是回答：

1. 这门课的核心问题是什么？
2. 课程大概分为哪些模块？
3. 模块之间是什么关系？
4. 哪些概念反复出现？
5. 哪些内容明显是考试重点？
6. 哪些图、算法、流程、公式是理解关键？

## 3. 生成课程总览

输出 `00_课程总览.md`。

课程总览必须包含：

- 课程定位
- 一句话学习目标
- 课程知识树
- 模块关系说明
- 模块重要程度与复习优先级
- 两周期末速成路径
- 文件覆盖说明
- 极限时间下的取舍建议

## 4. 建立模块-来源映射

在写模块手册前，先生成模块-来源映射。

推荐字段：

| 模块 | 来源文件 | 页码/幻灯片范围 | 核心概念 | 重要图示 | 典型例子 | 考试角度 |
|---|---|---|---|---|---|---|

该表可以作为 `source_map/module_source_map.md` 保存。

## 5. 模块级回读

这是本 Skill 的核心质量控制步骤。

对每个模块，在写文档前必须回读相关材料：

1. 打开相关 PDF/PPT 文件。
2. 重新检查页码/幻灯片范围。
3. 复核定义、公式、流程、图示和例子。
4. 对照模块知识树补充遗漏项。
5. 判断详略：基础概念简洁，核心机制详细，边缘内容略写。
6. 对无法确认的内容做“不确定”标注，不能伪造来源。

## 6. 生成模块知识手册

每个模块文档必须包含：

1. 模块定位
2. 本模块知识结构
3. 核心问题
4. 核心概念
5. 关键机制 / 算法 / 公式 / 流程
6. 典型实例
7. 重要图示
8. 易混淆知识点对比
9. 期末高频考点
10. 简答题式总结
11. 最后速记版
12. 来源定位与复核记录

## 7. 图示与实例处理

对于依赖图示或例子理解的内容，应单独列出。

每个图示条目推荐格式：

```markdown
### 图 1：<图示名称>

- 来源：<文件名>，第 <页/幻灯片> 页
- 作用：这张图解决了什么理解问题
- 如何读图：按层次/流程/箭头/组成部分解释
- 考试提示：这张图可能对应什么考法
```

如果可以导出图片，使用：

```markdown
![图示名称](images/module01_fig01.png)
```

如果无法导出图片，也要保留来源与解释。

## 8. 最终装配

生成最终文件夹：

```text
course_review_output/
├── README.md
├── 00_课程总览.md
├── 01_<模块>.md
├── 02_<模块>.md
├── images/
└── source_map/
    └── module_source_map.md
```

## 9. 完成前检查

完成前必须检查：

- 是否覆盖所有材料。
- 是否为每个模块执行了回读。
- 是否所有模块都有知识树。
- 是否重要图示和例子被单独列出。
- 是否明确区分来源内容与补充解释。
- 是否符合期末速成复习目标。
