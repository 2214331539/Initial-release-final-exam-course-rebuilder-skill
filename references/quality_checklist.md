# Quality Checklist

生成最终复习包前，使用本清单进行检查。

## A. 课程总览检查

- [ ] 是否覆盖了所有输入文件？
- [ ] 是否说明了这门课的核心问题？
- [ ] 是否生成了课程知识树？
- [ ] 是否说明模块之间的逻辑关系？
- [ ] 是否给出了模块复习优先级？
- [ ] 是否给出了两周期末速成路线？
- [ ] 是否说明了哪些内容可以略读？

## B. 模块文档检查

对每个模块检查：

- [ ] 是否在写作前回读了相关 PDF/PPT 页码或幻灯片？
- [ ] 是否包含模块定位？
- [ ] 是否包含本模块知识结构？
- [ ] 是否解释了核心问题？
- [ ] 是否整理了核心概念？
- [ ] 是否拆解了关键机制、算法、公式或流程？
- [ ] 是否列出典型实例？
- [ ] 是否列出重要图示？
- [ ] 是否比较了易混淆概念？
- [ ] 是否提取了期末高频考点？
- [ ] 是否提供了简答题式总结？
- [ ] 是否有最后速记版？
- [ ] 是否有来源定位与复核记录？

## C. 来源准确性检查

- [ ] 是否没有编造页码、图号或文件名？
- [ ] 是否把补充解释与课件原始内容区分开？
- [ ] 是否没有把通用教材知识误标为课程重点？
- [ ] 是否保留了重要图、表、例子、公式？
- [ ] 是否对不确定的考试重点做了说明？

## D. 期末复习适用性检查

- [ ] 学生是否能先读总览建立框架？
- [ ] 学生是否能按模块逐个击破？
- [ ] 文档是否避免了过长、过散、无重点？
- [ ] 是否突出理解路径，而不是只堆概念？
- [ ] 是否能直接服务于卷面答题？
