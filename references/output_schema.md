# Output Schema

本文件定义最终 Markdown 复习包的推荐结构。

## 1. 根目录结构

```text
course_review_output/
├── README.md
├── 00_课程总览.md
├── 01_<模块名称>.md
├── 02_<模块名称>.md
├── 03_<模块名称>.md
├── images/
│   ├── module01_fig01.png
│   └── module02_fig01.png
└── source_map/
    └── module_source_map.md
```

## 2. `00_课程总览.md` 必备章节

```markdown
# <课程名> 期末复习课程总览

## 1. 这门课到底学什么
## 2. 一句话学习目标
## 3. 课程核心问题链
## 4. 课程知识树
## 5. 模块关系与依赖
## 6. 模块优先级总表
## 7. 两周期末速成路线
## 8. 文件覆盖与来源说明
## 9. 极限时间取舍建议
```

## 3. 模块文档必备章节

```markdown
# <模块名>

## 1. 模块定位
## 2. 本模块知识结构
## 3. 核心问题
## 4. 核心概念
## 5. 关键机制 / 算法 / 公式 / 流程
## 6. 典型实例
## 7. 重要图示
## 8. 易混淆知识点对比
## 9. 期末高频考点
## 10. 简答题式总结
## 11. 最后速记版
## 12. 来源定位与复核记录
```

## 4. 模块优先级表

```markdown
| 模块 | 重要程度 | 难度 | 复习优先级 | 判断依据 |
|---|---|---|---|---|
| <模块名> | 高/中/低 | 高/中/低 | 最高/高/中/低 | <依据> |
```

## 5. 来源定位表

```markdown
| 知识点 | 来源文件 | 页码/幻灯片 | 复核状态 | 备注 |
|---|---|---|---|---|
| <知识点> | <文件名> | p.xx / slide xx | 已回读 | <备注> |
```

## 6. 图示条目

```markdown
### 图 <编号>：<图名>

- 来源：<文件名>，<页码/幻灯片>
- 对应知识点：<知识点>
- 图示作用：<说明>
- 如何理解：<分层解释>
- 考试提示：<可能考法>
```

## 7. 简答题式总结模板

```markdown
### 问：<可能的简答题问题>

答：

第一，<定义或背景>。
第二，<核心机制或步骤>。
第三，<优点/局限/适用场景>。
因此，<总结性结论>。
```
